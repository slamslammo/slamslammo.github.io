用Github和Hexo搭建个人博客笔记

Apr 2, 2018

1. 注册Github账号
要创建一个仓库（repo）和你的博客相关联。
使用Hexo的话，对这个repo的名字是有要求的，必须是：your_github_username.github.com格式。
最好生成并配置 SSH Keys。
也可以不配制，但不配置的话每次对自己的博客有改动要提交时，必须手动输入账号密码，配置了则不需要。

2. 安装git，node.js
下载并安装git：https://git-scm.com/download/win
下载并安装node.js：https://nodejs.org/en/

3. 安装并配置hexo
1）下载hexo：创建hexo文件夹，文件夹右击Git Bash Here，Git命令框中输入npm install -g hexo
2）在目录下初始化：hexo init
3）安装倚赖包：npm install

4. 部署博客
关联Github的部署，修改_config.yml配置博客
deploy:
type: git
repository: http://github.com/slamslammo/slamslammo.github.io.git
branch: master
本地发布：
hexo server (注：访问http://localhost:4000时，hexo server不可以按ctrl+c停止服务，否则404)
下载 Hexo 部署器：
npm install hexo-deployer-git –save
将博客部署到网上：
hexo deploy

5. 发布一篇博文
1）新建一篇文（本地博客文件夹的 /source/_post/ 目录下看到我们新建的 markdown 文件）：
hexo new “post_title”
2) 用 Markdown 编辑器打开文件进行编辑
3）使用 Hexo 生成静态网页，并发布到网上：
每次更新了博客，都需要 Hexo 重新生成一下静态网页，可以在 public 目录的相应日期下看到生成的文件。
hexo generate
hexo deploy
4）删除一篇博文
去本地博客文件夹的 /source/_post/ 目录下删除需要删除的 .md 文件。
去本地博客文件夹的 public 目录下删除该博文对应的文件夹（会按发布时间归到不同目录下）。
在终端重新生成静态网页并发布：
hexo generate
hexo deploy
5）当更新博客时发生了任何问题，可以在终端输入下述命令清理并重新生成静态网页：
hexo clean
hexo generate (注：缩写hexo g)
hexo deploy
